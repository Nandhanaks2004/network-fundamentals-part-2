The OSI Model

The Open Systems Interconnection model is a conceptual framework that standardizes network communication into 7 distinct layers. Click any layer to explore it in depth.

📦 Real-World Analogy: Think of it like a postal system — you write a letter (Application), translate it (Presentation), manage the session (Session), break it into packages (Transport), address them (Network), label them (Data Link), and physically deliver them (Physical).

📨 Data Flow: Sender → Receiver

When you type www.google.com in your browser, data travels DOWN through all 7 layers on your device (Encapsulation), then UP through all 7 layers on Google's server (Decapsulation). Each layer adds or removes its "header" like an envelope inside an envelope.

TCP/IP Model

TCP/IP is the practical model the internet actually uses — a simplified version of OSI with 4 layers. Every website you visit relies on this model.

OSI MODEL (7 Layers)

7. Application — HTTP, DNS, FTP
6. Presentation — SSL/TLS, JPEG
5. Session — NetBIOS, RPC
4. Transport — TCP, UDP
3. Network — IP, ICMP
2. Data Link — Ethernet, ARP
1. Physical — Cables, Wi-Fi

Encapsulation & Decapsulation

As data travels down the OSI layers, each layer wraps it with its own header — like envelopes inside envelopes. This is called Encapsulation. The reverse is Decapsulation.

🎁 Shipping Analogy: Your message is the gift. Transport adds the packing slip. Network adds the address label. Data Link adds the courier tag. Physical puts it on the truck as bits!

TCP Three-Way Handshake

Before any data is sent using TCP, both devices must agree to communicate. This three-step process establishes a reliable, ordered connection.

🚪 Door Knock Analogy: SYN = "Knock knock, anyone home?" → SYN-ACK = "Yes, come in!" → ACK = "Great, I'm stepping inside!" Only then does conversation begin.

SYN

Synchronize — Client says "I want to connect!" Sends its initial sequence number to sync communication.

SYN-ACK

Acknowledge — Server says "I'm ready!" Sends back its own sequence number + acknowledges client's.

ACK

Acknowledge — Client confirms "Got it! Let's start!" Connection is now established. Data flows.

TCP vs UDP

Two core transport protocols with very different personalities — one reliable and careful, one fast and reckless. Both have their place.

🔒 TCP — Transmission Control Protocol
📞 Like a Phone Call — Both sides confirm they're connected. If you miss a word, you say "Can you repeat that?"
  • Connection-oriented (3-way handshake)
  • Reliable — guarantees delivery
  • Ordered — packets arrive in sequence
  • Error checking + retransmission
  • Slower but dependable
Web Browsing Email File Download SSH
⚡ UDP — User Datagram Protocol
📣 Like a Loudspeaker — Broadcast your message and move on. You don't check who heard it.
  • Connectionless — no handshake
  • Fast — minimal overhead
  • No guarantee of delivery
  • No ordering or retransmission
  • Lightweight and efficient
Gaming Video Calls Live Streaming DNS

MAC Address vs IP Address

Every device has two identities on a network. One is permanent and physical, the other is logical and changeable.

🔖

MAC Address

AA:BB:CC:DD:EE:FF
  • Burned into your network card (NIC)
  • Never changes (permanent)
  • 48-bit hexadecimal
  • Used within local network (LAN) only
  • Layer 2 — Data Link
  • Like your fingerprint or VIN
Your permanent name — doesn't change no matter which network you join
🏠

IP Address

192.168.1.10
  • Assigned by DHCP (router) or manually
  • Changes per network
  • 32-bit (IPv4) or 128-bit (IPv6)
  • Used for global routing (internet)
  • Layer 3 — Network
  • Like your current home address
Your current address — changes when you move to a different network
Key Insight: When you take your laptop to a coffee shop, your MAC address stays the same, but your IP address changes. MAC = identity, IP = location. Routers use IP addresses to route between networks; switches use MAC addresses within a local network.

ARP Protocol

Address Resolution Protocol answers one critical question: "I know the IP address — but what's the MAC address of that device?" ARP bridges the gap between Layer 3 (IP) and Layer 2 (MAC).

🏢 Office Analogy: You know someone's employee ID (IP) but not their desk location (MAC). You shout "Who has employee ID 192.168.1.1?" — and that person calls back "It's me! My desk number is AA:BB:CC:DD:EE:FF"

dynamic vs static: Dynamic entries are learned automatically via ARP broadcasts. Static entries are manually set. The broadcast address (ff:ff:ff:ff:ff:ff) means "send to everyone on the LAN."

Subnetting & CIDR Notation

Subnetting divides a large network into smaller, efficient, manageable pieces. CIDR notation (/24, /26) tells us exactly how big each piece is.

🏢 Apartment Building Analogy: One big IP range = one big building. Subnetting = dividing the building into separate floors. Each floor (subnet) has its own set of apartments (hosts) but they're all under the same roof.

Routing & Default Gateway

Routing is the process of deciding where to send a packet. The default gateway is your network's exit door to the wider internet.

🚪 Exit Door Analogy: If you need to send a letter to someone in the same building, hand it to them directly. If they're in another city, take it to the Main Door (Default Gateway) and hand it to the mail carrier (router).

NAT — Network Address Translation

NAT allows multiple devices on your home network to share a single public IP address. Your router acts as a translator between private and public addresses.

📞 Office Receptionist Analogy: 10 employees use their own internal extensions (private IPs). Outside callers see only one main office number (public IP). When a reply comes in, the receptionist (router/NAT) checks the log and forwards it to the right employee.

STATIC NAT

One private IP maps to one specific public IP. Used for servers that need a permanent, known public address.

DYNAMIC NAT

Private IPs are assigned public IPs from a pool on demand. When done, the public IP goes back to the pool.

PAT (Overload)

Many private IPs share ONE public IP using different port numbers. This is what your home router uses!

Networking Tools

Every network professional uses these tools daily to diagnose, monitor, and understand network behavior. Learn what each tool does and why it matters.

📡
PING
ICMP Echo Request / Reply

Tests whether a device is reachable on the network and measures round-trip time. Uses ICMP protocol.

$ ping google.com
Why use it? Quick check: "Is this device alive?" Network admins run ping before anything else.
🗺️
TRACEROUTE
Hop-by-hop path tracing

Shows every router your packet passes through on its way to a destination. Uses TTL (Time-To-Live) trick.

← Click to trace
TTL Trick: Each packet starts with a TTL counter. Each router decrements it by 1. When it hits 0, the router reports back its IP — revealing the hop!
🔍
PING SWEEP
Network host discovery

Sends ping to every IP in a range to discover which devices are active on a network. Used by admins for inventory.

Admin use: "How many devices are on my network right now?" Ping sweep answers that instantly.
🦈
WIRESHARK
Network packet analyzer

Captures and inspects every packet on your network interface in real time. The gold standard for packet analysis.

  • Capture live network traffic
  • Inspect all protocol layers
  • Follow TCP streams
  • Analyze HTTP/DNS/ARP traffic
  • Export capture files (.pcap)
Why learn it? Wireshark proves that OSI layers are real — you can see every header, every IP, every MAC in each packet.
